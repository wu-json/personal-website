# RSS Feed for Signals

Add an RSS 2.0 XML feed at `/signals/feed.xml` that exposes Signals posts so readers can subscribe via feed readers.

## Goals

- **Build-time generation**: emit `build/signals/feed.xml` as part of `bun run build` (static, no runtime server).
- **RSS 2.0**: standard `<rss version="2.0">` with `<channel>`, `<item>` entries, plus `<content:encoded>` for full HTML bodies.
- **Signal entries**: one `<item>` per published Signal, newest first (id descending, matching list page).
- **Full-fidelity content**: each item carries the rendered markdown as HTML in `<content:encoded>`, plus a plain-text excerpt in `<description>`.
- **Absolute image URLs**: `<img src="/images/...">` paths in signal bodies are rewritten to `https://jasonwu.ink/images/...` so images render in feed readers.
- **Signals page icon**: a subtle, monochrome RSS icon/link beside the "Signals" heading that fits the site's pixel/glitch/terminal aesthetic.
- **Autodiscovery**: `<link rel="alternate" type="application/rss+xml">` in `index.html`.

## Non-goals

- JSON Feed or Atom (RSS 2.0 only).
- Podcast-specific tags / enclosures / media.
- Filtering or categories ‚Äî every Signal goes in the feed.

## Design

### File layout

| File                            | Purpose                                                                                             |
| ------------------------------- | --------------------------------------------------------------------------------------------------- |
| `src/plugins/rss.ts`            | Custom Vite plugin (`rssPlugin`) ‚Äî reads entries, generates XML, writes to `build/signals/feed.xml` |
| `vite.config.mts`               | Imports and adds `rssPlugin()` to the `plugins` array                                               |
| `index.html`                    | Adds `<link rel="alternate" type="application/rss+xml" ‚Ä¶>` autodiscovery tag                        |
| `src/screens/Signals/index.tsx` | Adds RSS icon/link beside the "Signals" heading                                                     |
| `build/signals/feed.xml`        | Output (gitignored; generated by build)                                                             |

### Plugin: `rssPlugin()`

Hooks into `writeBundle` (fires after the Vite bundle is written to disk ‚Äî perfect for emitting an extra static file). Lives at `src/plugins/rss.ts`. Registered in `vite.config.mts`:

```ts
import { rssPlugin } from './src/plugins/rss';

export default defineConfig({
  plugins: [react(), tailwindcss(), rssPlugin()],
  // ‚Ä¶
});
```

#### Plugin internals

1. **Read entries** ‚Äî `fs.readdirSync('src/screens/Signals/entries')`, filter `*.md`, read each file.
2. **Parse frontmatter** ‚Äî replicate the regex parser from `data.ts` (no import needed; it's 10 lines). Extract `id`, `timestamp`, `title`, `location`.
3. **Sort** ‚Äî by `id` descending (newest first).
4. **Limit** ‚Äî slice to 20 most recent Signals (keeps the feed file compact; plenty of headroom for years of posts at current cadence).
5. **Render markdown ‚Üí HTML** ‚Äî unified pipeline matching the runtime `MarkdownBody`:
   ```
   remark-parse ‚Üí remark-gfm ‚Üí remark-rehype (allowDangerousHtml) ‚Üí rehype-raw ‚Üí rehype-stringify
   ```
6. **Rewrite image URLs** ‚Äî a rehype plugin (or post-processing) that transforms `<img src="/images/...">` ‚Üí `<img src="https://jasonwu.ink/images/...">`.
7. **Plain-text excerpt** ‚Äî strip `<img>` tags and markdown syntax, normalize whitespace. First 300 chars for `<description>`.
8. **RFC-2822 dates** ‚Äî parse `timestamp` (format `YYYY.MM.DD // HH:MM:SS`) into a `Date`. If parsing fails (NaN), skip `<pubDate>` on that item.
9. **XML escaping** ‚Äî escape `&`, `<`, `>`, `"` in title and description text. Body goes into `<![CDATA[...]]>` ‚Äî no escaping needed.
10. **Write** ‚Äî `mkdirSync('build/signals', { recursive: true })`, `writeFileSync('build/signals/feed.xml', xml)`.

#### New dev dependencies

```
bun add -d unified remark-parse remark-gfm remark-rehype rehype-raw rehype-stringify
```

No runtime impact ‚Äî all used only inside the Vite plugin at build time.

### RSS feed structure

```xml
<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0" xmlns:content="http://purl.org/rss/1.0/modules/content/">
  <channel>
    <title>Jason Wu ‚Äî Signals</title>
    <link>https://jasonwu.ink/signals</link>
    <description>Jason's psychotic mind dump.</description>
    <language>en</language>
    <lastBuildDate>Mon, 28 Apr 2026 12:00:00 GMT</lastBuildDate>
    <item>
      <title>INITIAL BOOT SEQ</title>
      <link>https://jasonwu.ink/signals/001</link>
      <guid isPermaLink="true">https://jasonwu.ink/signals/001</guid>
      <pubDate>Sat, 14 Feb 2026 21:00:00 GMT</pubDate>
      <description>Changing the direction of this site‚Ä¶</description>
      <content:encoded><![CDATA[<p>Changing the direction of this site to be less of a resume shill‚Ä¶</p>]]></content:encoded>
    </item>
  </channel>
</rss>
```

#### Field behaviour

| Field               | Source                             | Fallback                         |
| ------------------- | ---------------------------------- | -------------------------------- |
| `<title>`           | `title` frontmatter                | `[id]` (e.g. `[001]`)            |
| `<link>` / `<guid>` | `https://jasonwu.ink/signals/{id}` | ‚Äî                                |
| `<pubDate>`         | `timestamp` ‚Üí RFC-2822             | omitted (no `<pubDate>` element) |
| `<description>`     | plain-text excerpt, ~300 chars     | empty string                     |
| `<content:encoded>` | full rendered HTML body            | empty CDATA                      |
| `<lastBuildDate>`   | current build time                 | ‚Äî                                |

### Icon on Signals page

Placed inline beside the `<h1>Signals</h1>` heading. Two viable approaches:

**A. Pixel-art SVG icon** ‚Äî a tiny custom RSS icon drawn to match `font-pixel` / `bio-glitch` vibe. 12√ó12 or 16√ó16 viewBox, monochrome `currentColor`, opacity `white/20` ‚Üí `white/50` on hover. Sits as an `<a>` wrapping the SVG.

**B. Monospace text link** ‚Äî `[rss]` or `// feed` in `text-white/20 text-[10px] font-mono`, aligned with the heading baseline, linked to `/signals/feed.xml`.

Either way, it should be quiet and understated ‚Äî not a bright orange RSS badge. The link opens normally (same tab, same origin ‚Äî it's an XML file, browsers render it natively).

### Link tag in `<head>`

Added to `index.html` alongside existing `<link>` tags:

```html
<link
  rel="alternate"
  type="application/rss+xml"
  title="Jason Wu ‚Äî Signals"
  href="/signals/feed.xml"
/>
```

This enables feed autodiscovery in browsers and feed readers.

## Implementation steps

1. **Install dev deps** ‚Äî `bun add -d unified remark-parse remark-gfm remark-rehype rehype-raw rehype-stringify`
2. **Create `src/plugins/rss.ts`** ‚Äî implement the Vite plugin with the pipeline described above
3. **Register plugin** ‚Äî import and add to `plugins[]` in `vite.config.mts`
4. **Add icon to Signals header** ‚Äî in `src/screens/Signals/index.tsx`, insert the RSS icon/link beside the `<h1>`
5. **Add autodiscovery `<link>`** ‚Äî in `index.html`
6. **Build and validate** ‚Äî `bun run build`, verify `build/signals/feed.xml` exists and is valid XML, spot-check against [W3C Feed Validator](https://validator.w3.org/feed/)
7. **Deploy** ‚Äî static file served at `https://jasonwu.ink/signals/feed.xml`

## Edge cases

| Scenario                                               | Handling                                                                 |
| ------------------------------------------------------ | ------------------------------------------------------------------------ |
| Signal has no `title`                                  | Use `[id]` (e.g. `[001]`)                                                |
| `timestamp` missing or unparseable                     | Omit `<pubDate>` for that item                                           |
| Signal body has no text (just images)                  | `<description>` may be empty; `<content:encoded>` still has `<img>` tags |
| `<img src="/images/...">` (relative URL)               | Rewrite to `https://jasonwu.ink/images/...` in the feed                  |
| Special XML chars in title/description (`&`, `<`, `>`) | XML-escape before insertion                                              |
| `build/signals/` directory doesn't exist               | `mkdirSync` with `recursive: true`                                       |
