# RSS Feed for Signals

Add an RSS 2.0 XML feed at `/signals/feed.xml` that exposes Signals posts so readers can subscribe via feed readers.

## Goals

- **Build-time generation**: emit `build/signals/feed.xml` as part of `bun run build` (static, no runtime server).
- **Dev-mode serving**: the feed is also served on-the-fly during `bun dev` via a Vite dev middleware, so clicking the link locally doesn't 404.
- **RSS 2.0**: standard `<rss version="2.0">` with `<channel>`, `<item>` entries, plus `<content:encoded>` for full HTML bodies.
- **Signal entries**: one `<item>` per published Signal, newest first (id descending, matching list page). Limited to 20 items.
- **Full-fidelity content**: each item carries the rendered markdown as HTML in `<content:encoded>`, plus a plain-text excerpt (~300 chars) in `<description>`.
- **Absolute URLs**: `<img src="/images/...">` and `<a href="/...">` paths in signal bodies are rewritten to `https://jasonwu.ink/...` so images and internal links work in feed readers.
- **Signals page link**: the `// live` subtitle row doubles as a clickable RSS link, with a `lucide-react` `Rss` icon next to the label. The entire row highlights on hover.
- **Autodiscovery**: `<link rel="alternate" type="application/rss+xml">` in `index.html`.

## Non-goals

- JSON Feed or Atom (RSS 2.0 only).
- Podcast-specific tags / enclosures / media.
- Filtering or categories — every Signal goes in the feed.

## Design

### File layout

| File                            | Purpose                                                                                |
| ------------------------------- | -------------------------------------------------------------------------------------- |
| `src/plugins/rss.ts`            | Custom Vite plugin (`rssPlugin`) — reads entries, generates XML, serves/writes feed    |
| `vite.config.mts`               | Imports and adds `rssPlugin()` to the `plugins` array                                  |
| `index.html`                    | Adds `<link rel="alternate" type="application/rss+xml" …>` autodiscovery tag           |
| `src/screens/Signals/index.tsx` | The `// live` subtitle row is a clickable `<a>` link to the RSS feed with a `Rss` icon |
| `build/signals/feed.xml`        | Output (gitignored; generated by build)                                                |

### Plugin: `rssPlugin()`

Lives at `src/plugins/rss.ts`. Registered in `vite.config.mts`:

```ts
import { rssPlugin } from './src/plugins/rss';

export default defineConfig({
  plugins: [react(), tailwindcss(), rssPlugin()],
  // …
});
```

Serves two modes via a shared `generateFeed()` async function:

**Build mode** (`writeBundle` hook): calls `generateFeed()` and writes the result to `build/signals/feed.xml`.

**Dev mode** (`configureServer` hook): registers a middleware at `/signals/feed.xml` that calls `generateFeed()` on each request and responds with `Content-Type: application/rss+xml`. This avoids 404s during local development without needing a separate build step.

#### Shared `generateFeed()` internals

1. **Read entries** — `fs.readdirSync('src/screens/Signals/entries')`, filter `*.md`, read each file.
2. **Parse frontmatter** — replicate the regex parser from `data.ts` (no import needed; it's 10 lines). Extract `id`, `timestamp`, `title`.
3. **Sort** — by `id` descending (newest first).
4. **Limit** — slice to 20 most recent Signals (keeps the feed file compact).
5. **Render markdown → HTML** — unified pipeline matching the runtime `MarkdownBody`:
   ```
   remark-parse → remark-gfm → remark-rehype (allowDangerousHtml) → rehype-raw → rehype-stringify
   ```
6. **Rewrite URLs** — string replacement after rendering: `/ src="\//g` → ` src="${BASE_URL}/"` and `/ href="\//g` → ` href="${BASE_URL}/"`. Fragment-only hrefs (`#...`) are left untouched.
7. **Plain-text excerpt** — strip `<img>` tags and markdown syntax, normalize whitespace. First 300 chars for `<description>`.
8. **RFC-2822 dates** — parse `timestamp` (format `YYYY.MM.DD // HH:MM:SS`) into a `Date` by replacing dots and separators: `new Date(ts.replace(/\./g, '-').replace(' // ', 'T'))`. If parsing fails (NaN), skip `<pubDate>` on that item.
9. **XML escaping** — escape `&`, `<`, `>`, `"` in title and description text. Body goes into `<![CDATA[...]]>` — no escaping needed.

#### New dependencies

```
bun add -d unified remark-parse remark-rehype rehype-stringify
bun add lucide-react
```

`remark-gfm` and `rehype-raw` were already present as transitive deps of `react-markdown`. `lucide-react` is a runtime dep (used for the `Rss` icon on the Signals page). The unified deps are dev-only.

### RSS feed structure

```xml
<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0" xmlns:content="http://purl.org/rss/1.0/modules/content/">
  <channel>
    <title>Jason Wu — Signals</title>
    <link>https://jasonwu.ink/signals</link>
    <description>Jason's psychotic mind dump.</description>
    <language>en</language>
    <lastBuildDate>...</lastBuildDate>
    <item>
      <title>INITIAL BOOT SEQ</title>
      <link>https://jasonwu.ink/signals/001</link>
      <guid isPermaLink="true">https://jasonwu.ink/signals/001</guid>
      <pubDate>Sat, 14 Feb 2026 21:00:00 GMT</pubDate>
      <description>Changing the direction of this site…</description>
      <content:encoded><![CDATA[<p>Changing the direction of this site to be less of a resume shill…</p>]]></content:encoded>
    </item>
  </channel>
</rss>
```

#### Field behaviour

| Field               | Source                             | Fallback                         |
| ------------------- | ---------------------------------- | -------------------------------- |
| `<title>`           | `title` frontmatter                | `[id]` (e.g. `[001]`)            |
| `<link>` / `<guid>` | `https://jasonwu.ink/signals/{id}` | —                                |
| `<pubDate>`         | `timestamp` → RFC-2822             | omitted (no `<pubDate>` element) |
| `<description>`     | plain-text excerpt, ~300 chars     | empty string                     |
| `<content:encoded>` | full rendered HTML body            | empty CDATA                      |
| `<lastBuildDate>`   | current build time                 | —                                |

### RSS link on Signals page

The `// live` subtitle below the "Signals" heading is a clickable `<a>` element linking to `/signals/feed.xml`. The entire row (`// live` text + `Rss` icon) is styled with `text-white/30 hover:text-white/50 transition-colors`, making the full width clickable and highlighting on hover.

The `Rss` icon from `lucide-react` is used — a proper quarter-circle arc RSS symbol. Sized 12×12px (`w-3 h-3`) with a `translate-y-[-1px]` nudge for visual alignment with the text baseline. The icon color matches the label text (`text-white/30`).

### Link tag in `<head>`

Added to `index.html` alongside existing `<link>` tags:

```html
<link
  rel="alternate"
  type="application/rss+xml"
  title="Jason Wu — Signals"
  href="/signals/feed.xml"
/>
```

This enables feed autodiscovery in browsers and feed readers.

## Edge cases

| Scenario                                               | Handling                                                                 |
| ------------------------------------------------------ | ------------------------------------------------------------------------ |
| Signal has no `title`                                  | Use `[id]` (e.g. `[001]`)                                                |
| `timestamp` missing or unparseable                     | Omit `<pubDate>` for that item                                           |
| Signal body has no text (just images)                  | `<description>` may be empty; `<content:encoded>` still has `<img>` tags |
| `<img src="/images/...">` or `<a href="/...">`         | Rewrite to `https://jasonwu.ink/...` in the feed                         |
| Special XML chars in title/description (`&`, `<`, `>`) | XML-escape before insertion                                              |
| Dev-mode access (no `build/` directory)                | Served on-the-fly via Vite dev middleware; no file needed                |
